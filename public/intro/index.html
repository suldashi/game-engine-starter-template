Lesson 2 - The basics of ECS

The Entity Component System (ECS) is an architectural pattern that is particularly suited to video game development. In this article we'll implement a basic example with multiple systems and components.

What is ECS?

In classic Object-oriented programming, a computer program is designed around objects, which contain all the data and logic needed for their operation. These objects are usually defined through classes, which serve as templates for creating many objects of the same type. To enable code reuse, classes may inherit from each other, taking on all the attributes and functionality of their parent class. However, this can lead to brittle design and deep levels of inheritance, making it difficult to reason about the inner workings of classes or extend them without risking breaking functionality somewhere else. In addition, multiple inheritance is very complex, making lateral code reuse difficult.

ECS takes a different approach from OOP. In ECS, all game objects are entities which are made up of multiple components attached to them that define their data and functionality. The behavior of these components is managed by various systems that interact in various ways to implement the rules of the video game. This way, we can attach components to entities at will, allowing us to create unique objects at runtime. In addition, the components are loosely coupled, making them easy to reason about, reuse, and modify as they only have a small amount of functionality built in.
